# ARTICLE VI — ARCHITECT (Summary)

Strategic problem decomposition before execution. Sits above APEX. Prevents 40-60% of wasted tokens by forcing precision thinking upfront.

| Rule | Name | Perspective | Gate |
|------|------|-------------|------|
| A1 | Concept Isolation | CS + Dev + Engineer | Formalize idea in one sentence |
| A2 | Scope Boundary | CS + Dev + Engineer | IN/OUT explicit |
| A3 | Constraint Extraction | CS + Dev + Engineer | Time, budget, hardware, API measured |
| A4 | Critical Path Question | CS + Dev + Engineer | Name ONE question that kills project |
| A5 | Dependency Questions | CS + Dev + Engineer | Does each dep exist? Work? Alternatives? |
| A6 | Architecture Questions | CS + Dev + Engineer | Layers, call graph, break points |
| A7 | Jargon Extraction | CS + Dev + Engineer | Replace English with formal notation |
| A8 | Success Criteria Language | CS + Dev + Engineer | Boolean predicates, all testable |
| A9 | Failure Mode Language | CS + Dev + Engineer | Precise failure states, resource causes |
| A10 | Proof-of-Concept Scope | CS + Dev + Engineer | Minimal test, 10% of budget |
| A11 | Library Suitability Matrix | CS + Dev + Engineer | 2x2: exists? solves problem? |
| A12 | Alternative Paths | CS + Dev + Engineer | 2+ alternatives, score all, choose best |
| A13 | Phase Token Budget | CS + Dev + Engineer | Estimate per phase, total, buffer |
| A14 | Dead End Detection Budget | CS + Dev + Engineer | Reserve 10% to catch dead ends early |
| A15 | Checkpoint Compression Cost | CS + Dev + Engineer | E25 savings measured, applied to budget |
| A16 | Two-Iteration Threshold | CS + Dev + Engineer | Two fails = dead end, stop |
| A17 | Assumption Risk Ranking | CS + Dev + Engineer | Rank by consequence, test critical first |
| A18 | Escape Hatch Criteria | CS + Dev + Engineer | When do you stop? Define termination. |
| A19 | Definition of Done | CS + Dev + Engineer | All 7+ predicates true = done |
| A20 | Decomposition Document | CS + Dev + Engineer | Living specification, update as learn |

**Core principle:** Five minutes of ARCHITECT prevents five hours of APEX iteration.
