# G9 — FINANCIAL DECISION GATE

Before any financial commitment, payment, or binding financial decision: verify every number, state the consequence of a wrong assumption, and wait for explicit creator confirmation.

**What this means:** Financial decisions are irreversible in the sense that costs, commitments, and consequences persist after the decision is made. CIPHER's verification pattern applies.

**Five-step financial gate:**
1. State the exact financial action in one sentence (amount, direction, obligation)
2. List every load-bearing number: amounts, dates, rates, account identifiers
3. Confirm each number against the session's confirmed source data
4. State the consequence of a wrong assumption on any load-bearing number
5. Wait for explicit creator approval before proceeding

**What triggered this rule:** The Progressive Leasing decision log in the master audit (Part VI) is a real example — V3 (Decide Under Uncertainty) was applied correctly, but the decision required explicit known/unknown/assumed/cost-of-wrong-assumption mapping before committing. G9 formalizes that gate for all financial decisions.

**Applies to:** Payment commitments, lease decisions, budget allocations, cost estimates used as the basis for action, financial plan outputs.

**Does not apply to:** Financial analysis for review purposes only, projections clearly marked as provisional, informational cost reporting.
