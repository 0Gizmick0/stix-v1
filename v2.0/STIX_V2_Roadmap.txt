4. VERSION DECISION
This is Version 2. Not Version 1.2.
The threshold from patch to rebuild is crossed when the change requires:

New governing values (three added: TRANSPARENCY, SAFETY, EFFICIENCY)
New top-level protocols that govern existing protocols (RELAY governs CIPHER)
Infrastructure layers that do not exist anywhere in V1.0 (memory architecture, trust calibration, output lifecycle, signal classification)
A fundamentally different compliance model (OBSERVE makes the stack self-policing — V1.0 is user-policed)
New appendices that define error handling and operating cadence (V1.0 has neither)

Version 1.2 would be: adding rules to existing articles, expanding CIPHER in place, adding boundary entries.
Version 2 is: promoting CIPHER into a sub-protocol under a new top-level layer, adding three new governing values, adding four new operational protocols, adding six new infrastructure layers, adding four new appendices, and changing the fundamental compliance model from advisory to self-policing.
That is a rebuild, not a patch.

6. RISK OF DOING IT WRONG
If this is handled as a series of patches rather than a controlled rebuild:
Rule contradiction. RELAY generalizes CIPHER. If CIPHER is patched in place while RELAY is added separately, their jurisdiction overlaps. Two rules can fire simultaneously on the same action with conflicting instructions.
Governing value collision. Adding TRANSPARENCY, SAFETY, and EFFICIENCY without rebuilding the binding sequence produces an undefined priority order when values conflict. Which value wins when OBSERVE (TRANSPARENCY) and RISK (SAFETY) both fire on the same event is not defined.
Compliance model mismatch. V1.0 is user-policed. V2 is self-policing. Patching OBSERVE into V1.0 without rebuilding the session flow produces a hybrid where some outputs are compliance-checked and others are not. That is worse than no compliance checking because it creates false confidence.
Provenance corruption. A patched framework carries no clear version boundary. Anyone using the framework cannot distinguish which rules are V1.0 confirmed from which are unconfirmed V2 additions. F11 (preserve provenance always) cannot be satisfied in a patched document.
Audit trail failure. The 39-page master audit documents V1.0. A patched V1.0-plus document has no audit trail. V2 needs its own audit document with its own decision log.
